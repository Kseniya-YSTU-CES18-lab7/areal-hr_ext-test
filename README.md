# areal-hr_ext-test
Реализовать веб-приложение, в котором специалист по кадрам ведет учет сотрудников в нескольких организациях.

Задания 1-й недели:
Схема БД в Draw.io (3НФ)
Структура проекта
Описание инструментария
Информация о работе с Git

# Инструментарий

Операционная система - Windows 10 + Docker Desktop (WSL2)
IDE Visual Studio Code
PostgreSQL - Версия 17, запускается в Docker контейнере
Docker - Docker Desktop с WSL2 backend

# Используемые технологии

Frontend: Vue.js 3.5, Quasar, Vite
Backend: Node.js 22, NestJS 10
База данных: PostgreSQL 17
Контейнеризация: Docker / Docker Compose

# Мягкое удаление

Во всех таблицах, где требуется CRUD, реализовано мягкое удаление через поле `Удаление`. Запись не удаляется физически, только помечается как удаленная. Исключение — таблица `История_изменений` (т.к логи, не удаляются).

# Неделя 2: Реализация CRUD — Организации, Отделы, Должности

# Backend (NestJS)
- Проект инициализирован в папке `api/`
- Подключение к БД: TypeORM + PostgreSQL 17 (Docker) + ConfigService
- Миграции: `node-pg-migrate` с конфигурацией в `pg-migrate.config.json`

# CRUD API

- Organizations: POST/GET/PATCH/DELETE /organizations — валидация, дубликаты, soft delete
- Departments: POST/GET/PATCH/DELETE /departments + GET /departments/tree — вложенная структура, дерево отделов, soft delete
- Positions: POST/GET/PATCH/DELETE /positions — уникальность названия, soft delete

# Тестирование
- Файл: [`api/http/organizations.http`](api/http/organizations.http)
- Инструмент: VS Code REST Client
- Покрытые сценарии: создание, чтение, обновление, мягкое удаление, проверка дубликатов

# Запуск

```bash
# 1. Запустить БД
docker-compose up -d db

# 2. Применить миграции
cd api
npm run migration:up

# 3. Запустить backend
npm run start:dev

# 4. Тестировать API
# Открыть api/http/organizations.http в VS Code → "Send Request"
```

# ИСПРАВЛЕНИЯ ПО 2-Й НЕДЕЛЕ:
1. Два инструмента миграций - Оставлен только `node-pg-migrate`, удалён конфиг миграций TypeORM
2. Захардкоженные креды вынесены в `.env`, добавлен `.env.example`, пароли убраны из кода
3. UNIQUE + soft delete конфликт - Частичный уникальный индекс: `WHERE deleted_at IS NULL`
4. `console.log` в коде заменён на `Logger` из `@nestjs/common`
5. Слабая валидация DTO - Добавлены `@IsNotEmpty` и `@Transform` для обрезки пробелов
6. Мёртвый код `restore()` - Добавлены эндпоинты `POST /:id/restore` для всех сущностей
7. Непоследовательный `async/await` - Унифицирован стиль: все методы контроллеров теперь `async/await`


# Неделя 3. Реализовано: CRUD — Сотрудники, Файлы, Кадровые операции, История

Employees — CRUD API для сотрудников (UUID, паспортные данные, адреса, файлы)
Passports — паспортные данные (связь 1:1 с сотрудником)
Addresses — адреса регистрации (связь 1:1 с сотрудником)
Files — метаданные файлов (связь 1:n с сотрудником)
HrOperations — кадровые операции: приём, изменение зарплаты, перевод, увольнение
History — логирование изменений (полиморфная связь)

# Особенности

Мягкое удаление (deletedAt)
Валидация через class-validator + class-transformer
camelCase в коде → snake_case в БД
UUID у сотрудников, SERIAL у остальных
Связи через @OneToOne, @ManyToOne, @OneToMany
