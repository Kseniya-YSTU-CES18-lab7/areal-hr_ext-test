# areal-hr_ext-test
Реализовать веб-приложение, в котором специалист по кадрам ведет учет сотрудников в нескольких организациях.

Задания 1-й недели:
Схема БД в Draw.io (3НФ)
Структура проекта
Описание инструментария
Информация о работе с Git

# Инструментарий

Операционная система - Windows 10 + Docker Desktop (WSL2)
IDE Visual Studio Code
PostgreSQL - Версия 17, запускается в Docker контейнере
Docker - Docker Desktop с WSL2 backend

# Используемые технологии

Frontend: Vue.js 3.5, Quasar, Vite
Backend: Node.js 22, NestJS 10
База данных: PostgreSQL 17
Контейнеризация: Docker / Docker Compose

# Важная информация в структкре

Файл `.env` сПеременные окружения (реальные пароли...) не коммитится (добавлен в `.gitignore`)
Папки `node_modules/` не коммитятся — устанавливаются через `npm install`
Для запуска используется `.env.example` как шаблон: скопировать в `.env` и заполнить значения

# Состав схемы (11 таблиц)

Таблица - Назначение - Особенности:

Организации - Компании
Отделы - Структурные подразделения - Вложенная структура через `Код родителя`
Должности - Справочник должностей
Сотрудники - Личные данные
Паспортные данные - Паспортные данные - Связь 1:1 с сотрудником
Адрес регистрации - Адрес регистрации - Связь 1:1 с сотрудником
Файлы - Сканы паспортов и документы - Связь с пользователем, который загружает
Сотрудники файлы - Связь сотрудников и файлов = Составной ключ, связь n:n
Кадровые операции - Прием, переводы, изменения, увольнение - Хранит старые и новые значения
История изменений - Логирование всех действий - Полиморфная связь
Пользователи - Администраторы и менеджеры

# Связи между таблицами

Тип связи - Где используется:

1 - n  Организации - Отделы, Сотрудники - Кадровые операции, Отделы - Кадровые операции, Должности - Кадровые операции, Сотрудники - Сотрудники файлы, Файлы - Сотрудники файлы, Пользователи - История изменений, Пользователи - Файлы
1 - 1 Сотрудники - Паспортные данные, Сотрудники - Адрес_регистрации
Самосвязь 1 - n Отделы - Отделы (через `Код родителя` для вложенности)
Полиморфная (n → 1) История изменений - Организации, Отделы, Должности, Сотрудники, Кадровые 
операции (пунктирные линии)

# Мягкое удаление

Во всех таблицах, где требуется CRUD, реализовано мягкое удаление через поле `Удаление`. Запись не удаляется физически, только помечается как удаленная. Исключение — таблица `История_изменений` (т.к логи, не удаляются).

# Работа с Git

# Используемые инструменты

GitHub Desktop - Основная работа: коммиты, пуши, просмотр истории (визуально)
Терминал VS Code - Быстрые команды: `git status`, `git log`...
Браузер - Проверка, что изменения успешно отправлены на GitHub

# Частые команды

```bash
# Проверить, какие файлы изменены
git status

# Добавить все изменения в коммит
git add .

# Создать коммит с сообщением (делаю через GitHub Desktop, но можно и через терминал)
git commit -m "тип: краткое описание"

# Отправить изменения на сервер
git push origin main

# Посмотреть историю коммитов
git log --oneline

# Создать новую ветку
git checkout -b feature/название

# Переключиться на ветку main
git checkout main

# Слить ветку с текущей
git merge feature/название
```
